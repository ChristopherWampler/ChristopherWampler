# Christopher Wampler

I build systems that solve real problems for real people.

Currently focused on **AI-integrated applications**, **automation**, and **full-stack product development** with a bias toward things that actually ship and run in production.

## What I'm Building

**[PAI-OS](https://github.com/ChristopherWampler/PAI0S)** - A local-first multi-agent personal operations system built around deterministic health and weather pipelines, event journaling, projection-driven read models, and optional LLM orchestration. Designed to showcase systems engineering, data flow design, failure boundaries, and reproducible Docker-based demos.

**[Slarti](https://github.com/ChristopherWampler/Slarti)** - A compound AI system built as a family garden companion. Multi-provider AI routing, pgvector semantic memory, Discord plus voice integration, automated weather monitoring, and a regional knowledge engine with 390+ indexed sources from MU Extension and the Farmer's Almanac. Runs in production on a home server with 6 automated agents on cron.

**[Homeschool App](https://github.com/ChristopherWampler/Homeschool_App)** - Full-stack PWA for homeschool administration. React 18 / TypeScript frontend, PostgreSQL backend, n8n automation, offline-first architecture, and AI-powered weekly insights. Built for Missouri state compliance. Design principle: *if it's not easier than Excel, it failed.*

## How I Think

I care about **systems design** - how components fit together, how data flows, and how things fail gracefully. I document decisions, not just code. I build for maintainability because I'm usually the one maintaining it at 3 AM.

I am drawn to problems where the solution is not another SaaS dashboard but a thoughtful integration of real tools into real workflows. I care about clear service boundaries, explicit data ownership, and keeping deterministic logic separate from AI-generated behavior.

**Core stack:** Python, TypeScript, React, PostgreSQL, Docker, Claude / Gemini / OpenAI APIs
